Discovering tables in Manifold via ODBC

Michael Sumner

20 January 2016

In this Project “Components.Map” I have a number of components, to try different kinds of queries. I am using R with the CRAN package RODBC and my in-development package manifoldr.

library(RODBC)  ## install.packages("RODBC")
library(manifoldr) ## https://github.com/mdsumner/manifoldr
mapfile <- "C:/temp/Components.map"
con <- odbcConnectManifold(mapfile)
print(con)  ## summary of the connection
## RODBC Connection 2
## Details:
##   case=nochange
##   DRIVER={Manifold Project Driver (*.map)}
##   DBQ=C:\temp\Components.map
##   Unicode=True
##   Ansi=True
##   OpenGIS=True
##   DSN=Default

If I ask what tables are available, it’s not comprehensive.

sqlTables(con)
##   TABLE_CAT TABLE_SCHEM                TABLE_NAME TABLE_TYPE REMARKS
## 1      <NA>        <NA>                 LoneTable      TABLE        
## 2      <NA>        <NA>         DrawingOwnedTable      TABLE        
## 3      <NA>        <NA>              QuerySurface       VIEW        
## 4      <NA>        <NA>          GEOMETRY_COLUMNS      TABLE        
## 5      <NA>        <NA> SPATIAL_REFERENCE_SYSTEMS      TABLE        
## 6      <NA>        <NA>              drawing_attr       VIEW        
## 7      <NA>        <NA>        drawing_wkb_native       VIEW        
## 8      <NA>        <NA>        drawing_wkb_latlon       VIEW

I can read from LoneTable, DrawingOwnedTable, and QuerySurface unsurprisingly.

sqlQuery(con, "SELECT * FROM [LoneTable]")
##   Column
## 1      1
sqlQuery(con, "SELECT [ID], [X (I)], [Latitude (I)] FROM [DrawingOwnedTable]")
##   ID X (I)  Latitude (I)
## 1  1  -196  0.0017606980
## 2  3  -338 -0.0006378039
## 3  5   -44 -0.0014013718
## 4  7   104  0.0013115403
sqlQuery(con, "SELECT [Height (I)], [Center Y (I)] FROM [QuerySurface]")  ## SELECT TOP 1 * [From QuerySurface]
##   Height (I) Center Y (I)
## 1          0        199.5

But I can also read from Drawing and Surface.

sqlQuery(con, "SELECT [ID] FROM [Drawing]")
##   ID
## 1  1
## 2  3
## 3  5
## 4  7
sqlQuery(con, "SELECT TOP 2 [Easting (I)] FROM [Surface]")
##   Easting (I)
## 1           0
## 2           1

This is fine except that I can’t programmatically get access to the existence of Surface via ODBC, whereas with the Drawing, Query and stand alone Table I only need list them from sqlTables and query the column names in turn to get a summary of the contents in the .map file.

Is this a limitation of the ODBC interface, or one of the RODBC package, or one of Manifold’s ODBC driver?

Thanks!

odbcClose(con)
